## 1. 项目目标

打造一个简洁、现代、易维护的个人产品展示网站，用于对外展示产品能力与案例，支持后续快速迭代为多页面版本。

### 1.1 核心目标

1. 清晰展示「我是谁 + 我做过什么产品 + 如何联系我」
2. 提供可扩展的信息结构，支持后续新增项目与文章
3. 首屏加载快、移动端友好、基础 SEO 完整

## 2. 目标用户

1. 招聘方 / 面试官
2. 潜在合作方 / 客户
3. 同行开发者与设计师

---

## 3. 版本范围（v-0.1.0）

### 3.1 页面范围

当前版本采用单页结构（Landing Page），包括：

1. Hero（个人介绍 + 价值主张）
2. 产品项目列表（卡片）
3. 关于我（能力、技术栈）
4. 联系方式（邮箱/社媒链接）

### 3.2 功能范围

1. 项目卡片展示（标题、简介、标签、链接）
2. 基础响应式布局（桌面 + 移动端）
3. 导航锚点跳转
4. 基础 SEO（title、description、Open Graph）

---

## 4. 信息架构

1. Header
   - Logo/名称
   - 导航项：项目、关于、联系
2. Main
   - Section: Hero
   - Section: Projects
   - Section: About
   - Section: Contact
3. Footer
   - 版权信息
   - 外部链接

---

## 5. 设计与体验要求

1. 整体设计语言：对齐 Apple Human Interface Guidelines 的审美与交互原则
2. 风格：简洁、现代、克制，强调内容与留白
3. 颜色：低饱和中性色为主，谨慎使用强调色
4. 排版：明确标题层级（H1/H2/H3），注重可读性与节奏
5. 交互：过渡自然、反馈克制，避免炫技式动画
6. 可访问性：对比度、键盘可达、语义化标签

---

## 6. 技术方案

- **框架**: Next.js (App Router)
- **样式**: Tailwind CSS
- **语言**: TypeScript
- **代码规范**: ESLint
- **运行方式**: npm scripts

## 7. 验收标准（DoD）

1. 本地可运行：`npm run dev` 正常
2. 主要版块可访问且样式完整
3. 移动端与桌面端无明显布局错乱
4. Lighthouse（基础目标）
   - Performance >= 80
   - Accessibility >= 90
   - Best Practices >= 90
   - SEO >= 90
5. 文档与代码已提交到 `v-0.1.0` 分支

## 9. 风险与应对

1. 内容素材不足 → 先用占位数据，后续替换
2. 视觉方向反复 → 先确定 1 套设计 token 再开发
3. 迭代追溯困难 → 版本化 PRD 文件命名并持续沉淀
