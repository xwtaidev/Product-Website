## P1 - 应做（提升核心体验）

- [ ] 新增博客详情页面（Blog Post Detail）
  - 说明：在现有博客列表基础上补齐详情页模板，支持通过 slug 访问单篇内容，并具备基础 SEO 与可读性排版。
  - 影响范围：`src/app/blog` 路由结构、详情页模板、文章数据源（本地 md/json 或 TS 数据）。
  - 验收标准：
    - 支持 `/blog/[slug]` 访问单篇文章；
    - 博客列表可正确跳转到对应详情页；
    - 详情页包含标题、日期、正文、返回入口；
    - 每篇文章具备独立 metadata（title/description）；
    - 移动端阅读体验正常（字号、行高、段落间距可读）。

- [ ] 优化首页 / 博客列表 / 工作列表的 Hero 布局一致性
  - 说明：统一三类页面 Hero 的版式规则，解决当前标题、描述、操作按钮、间距在不同页面不一致导致的“松散/拥挤”问题。
  - 影响范围：`src/app/page.tsx`、`src/app/blog/page.tsx`、`src/app/work/page.tsx`（若文件路径不同以实际为准）。
  - 验收标准：
    - 三个页面 Hero 在桌面端与移动端均保持一致的布局节奏（标题层级、最大宽度、段落行长、按钮位置）；
    - 首屏不出现内容挤压、留白失衡、按钮换行异常；
    - 在 375 / 768 / 1024 / 1440 断点下视觉对齐良好；
    - Hero 区支持后续复用（抽成可复用组件或统一样式规范）。
